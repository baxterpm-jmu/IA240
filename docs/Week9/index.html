Week 9: Machine Learning and Natural Language Processing

IA 240: Technology Applications in a Networked World

Machine learning has moved from research demonstrations to operational deployment across the intelligence community and military. This lecture covers the theoretical foundations of ML, core algorithms and methods, natural language processing, and the real-world intelligence applications — and failures — that define the current state of AI in the IC.

How the intelligence community is deploying generative AI — including a CIA-built tool, Osiris, that summarizes open-source data for thousands of analysts — while wrestling with hallucination. The CIA’s CTO calls the models a “crazy, drunk friend.”

A former quant on how models that are opaque, unaccountable, and optimized for the wrong target cause real harm at scale.

As you read, consider:

  • O’Neil says a model encodes the goals and values of its builder. For a conflict- or threat-prediction model, whose goals get encoded, and how would you audit for that?
  • What makes a model a “weapon” in her terms — and which of those features could be fixed by better evaluation rather than better data?

Part I: What is Machine Learning?

Defining Machine Learning

Herbert Simon’s definition frames the concept simply: “Learning is any process by which a system improves performance from experience.”

Tom Mitchell’s formal definition from 1998 provides a precise framework:

A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E.1

This means a well-defined ML problem specifies:

  • T — the task (e.g., classify emails as spam or not-spam)
  • P — how we measure success (e.g., accuracy, precision, recall)
  • E — the training data the system learns from

Types of Machine Learning

Supervised Learning

The model is trained on labeled examples — pairs of input and correct output. It learns to map inputs to outputs. Examples: spam detection, image classification, conflict onset prediction, sentiment analysis.

Unsupervised Learning

No labels. The model discovers structure in data on its own — clusters, associations, patterns. Examples: customer segmentation, anomaly detection, topic modeling, detecting unusual financial transactions.

Reinforcement Learning

An agent interacts with an environment and learns through trial and error, receiving rewards for good actions and penalties for bad ones. Examples: game-playing AIs, autonomous systems, adaptive control.

Semi-Supervised / Self-Supervised

Combines small amounts of labeled data with large amounts of unlabeled data. Increasingly common in modern large language models.

Large Language Models (LLMs)

Large language models like GPT-4, Claude, and Gemini are built on a specific ML architecture — the transformer — trained on massive text corpora.

Training Data
LLMs are trained on text scraped from the web, books, code repositories, and other sources — often hundreds of billions of tokens. The quality and composition of training data shapes the model’s knowledge, biases, and capabilities.
Architecture (Transformer)
Attention mechanisms allow the model to weigh the relevance of every token in context when generating each next token. This is what enables coherent long-form text generation.
Pre-training and Fine-tuning
The model first learns statistical patterns from massive text data (pre-training), then is further trained on smaller curated datasets to align it with specific tasks or human preferences (fine-tuning via RLHF — Reinforcement Learning from Human Feedback).
Hallucination
LLMs generate confident-sounding text even when the generated content is factually incorrect. This is not a bug to be patched — it is an inherent property of next-token prediction. For intelligence analysis, this makes LLM output a starting point for investigation, not a source.
Context Window
Within a given conversation or document, the model uses all prior tokens as context. Longer context windows enable analysis of longer documents, but LLMs have no access to information beyond their training cutoff without retrieval augmentation.
WarningLLMs in Intelligence Contexts

Speed and volume: AI can process far more text than human analysts. Bias and hallucination: LLMs generate confident-sounding text even when wrong. Adversarial use: adversaries also have access to AI — generative AI changes influence operations, disinformation, and synthetic media. Classification constraints: most LLMs train on open data, constraining IC use.

Part II: ML Methods and Algorithms

Common Algorithms

Decision Trees

A flowchart-like structure that partitions data recursively based on feature values. Each internal node tests a feature; each leaf node assigns a class or value. Decision trees are highly interpretable but prone to overfitting.

K-Nearest Neighbors (k-NN)

Classifies new data points based on majority vote among the \(k\) most similar training examples. Simple and intuitive; performance degrades in high-dimensional spaces (“curse of dimensionality”).

Support Vector Machines (SVM)

Finds the hyperplane in feature space that maximally separates classes. Kernel methods extend SVMs to non-linear decision boundaries by mapping data to higher-dimensional spaces.

Neural Networks

Composed of interconnected neurons organized in layers. Learns through backpropagation — repeatedly adjusting weights to minimize prediction error using gradient descent. Key architectures:

  • Convolutional Neural Networks (CNNs): designed for spatial data (images); used in imagery exploitation
  • Recurrent Neural Networks (RNNs) / LSTMs: designed for sequential data (text, time series); used in NLP and SIGINT analysis
  • Transformers: the architecture underlying modern LLMs; attention mechanism models long-range dependencies

Ensemble Methods

Random Forest (Bagging): Trains multiple decision trees on different random subsets of training data; combines predictions by majority vote. Reduces overfitting. Gradient Boosting (Boosting): Sequentially trains models where each focuses on examples the previous model got wrong. Examples: XGBoost, LightGBM — widely used in IC-adjacent analytical competitions.

K-Means Clustering (Unsupervised)

Partitions data into \(k\) clusters by minimizing the sum of squared distances from each point to its cluster centroid. Applications: identifying distinct behavioral groups in financial transaction data; segmenting states by foreign policy orientation.

The Bias-Variance Tradeoff

High Bias (Underfitting)

Model is too simple to capture the underlying pattern. Performs poorly on both training and test data.

Example: fitting a straight line to data with a curved relationship.

High Variance (Overfitting)

Model is too complex — it fits the training data (including its noise) too closely. Performs well on training data, poorly on new data.

Example: a deep decision tree that memorizes every quirk of the training set.

The optimal model minimizes total error = Bias² + Variance + Irreducible Error. Regularization methods (L1/L2 penalties, dropout in neural networks) control variance; more complex models reduce bias.

Part III: Sentiment Analysis

What is Sentiment Analysis?

Sentiment analysis is the computational process of identifying and extracting subjective information — opinions, attitudes, emotions — from textual data. It transforms unstructured text into quantifiable sentiment dimensions.

Applications in intelligence: monitoring foreign government media sentiment toward the United States; tracking social media responses to policy announcements; analyzing extremist forum content for radicalization signals.

Processing Pipeline

Text Preprocessing
Clean raw text: tokenization (splitting text into words/subwords), removing stop words, stemming or lemmatization (reducing words to root form), correcting spelling/encoding errors.
Feature Extraction
Convert preprocessed text into numerical features. Methods: bag-of-words (count of each word), TF-IDF (term frequency–inverse document frequency), word embeddings (Word2Vec, GloVe), and contextual embeddings (from transformer models like BERT).
Classification / Scoring
Apply a model to assign sentiment scores or categories. Rule-based methods use lexicons of pre-scored words. Machine learning methods train classifiers on labeled datasets. Hybrid approaches combine both.

Key Sentiment Dimensions

Polarity
The direction of sentiment: positive, negative, or neutral. The most commonly reported sentiment dimension.
Subjectivity
The degree to which text expresses opinion vs. fact.
Emotion
More granular than polarity: anger, joy, sadness, fear, surprise, disgust. Useful for understanding the intensity and type of reactions to events.
Intensity
The magnitude of the expressed sentiment. “Furious” has higher negative intensity than “unhappy.”
WarningLimitations of Sentiment Analysis

Sentiment analysis tools built on English-language training data perform poorly on other languages, dialects, or specialized registers. Sarcasm and irony are notoriously difficult to detect algorithmically. Domain adaptation (fine-tuning on relevant corpora) and contextual human review remain essential for analytical reliability in intelligence applications.

Part IV: Artificial Intelligence in Intelligence Operations

From Theory to Practice — Real Deployments

AI and ML are no longer future capabilities in the intelligence and defense context. They are deployed operational tools with real consequences.

Project Maven (2017–present)
The Pentagon’s Algorithmic Warfare Cross-Functional Team, launched in April 2017. Initially contracted with Google (contract not renewed in 2018 after employee protests), then moved to Palantir and Anduril. Maven applies computer vision to drone and surveillance video to automatically detect, identify, and track objects and persons of interest. As of 2023, it operates across multiple combatant commands.
NSA SKYNET (disclosed 2015)
An NSA program, revealed in documents released by The Intercept, that applied machine learning to metadata from Pakistani telecommunications networks to identify potential al-Qaeda couriers — based on call patterns, phone switching behavior, and travel. Human rights researchers raised concerns that the model’s false positive rate may have contributed to drone strike targeting errors.
Palantir Gotham
A data integration and analysis platform used by the CIA, FBI, NSA, and military intelligence. It fuses data from disparate sources — financial records, communications metadata, geospatial data, biometrics — into a unified analytical environment. It is one of the most widely deployed data science platforms in the IC.
DARPA SEMAFOR
A program to detect machine-generated text, images, and video. As generative AI makes synthetic media creation trivially easy, the IC requires tools to identify AI-generated content — propaganda, fabricated imagery, synthetic audio — in the information environment.
GPT-4 and IC Experimentation
In 2023, the CIA announced development of an internal LLM-based tool for analysts to query open-source intelligence. The Office of the Director of National Intelligence (ODNI) published a framework for responsible AI use in the IC in 2023, acknowledging that commercial LLMs were already being explored across multiple agencies.

Deepfakes, Synthetic Media, and Intelligence

Generative AI enables the creation of synthetic audio, video, and images that are increasingly indistinguishable from authentic content — often called “deepfakes.”

Threat to Intelligence Analysis

  • Fabricated evidence: synthetic imagery or video can be created to support false claims about events that did not occur (e.g., fabricated atrocity footage)
  • Identity fraud: audio clones of senior officials can be used in vishing attacks or to impersonate commanders
  • Disinformation at scale: AI allows influence operations to produce personalized, localized content at a volume no human team could match
  • Provenance challenge: when authentic and synthetic media are mixed, the credibility of all visual evidence comes into question

Detection and Countermeasures

Current detection methods include:

  • Forensic analysis: GAN-generated images often contain subtle artifacts (facial asymmetry, inconsistent lighting, ear/hair rendering errors)
  • Metadata analysis: file metadata, compression signatures, and upload provenance can indicate synthetic origin
  • Cryptographic provenance: initiatives like the Content Authenticity Initiative (CAI) and C2PA standard attach cryptographic signatures to authentic media at the point of capture
  • Behavioral anomaly: synthetic video sometimes fails on subtle physics — hair movement, light reflection on eyes, blinking patterns
WarningThe Liar’s Dividend

Researchers Chesney and Citron (2019) coined the term “liar’s dividend” to describe a secondary effect of deepfake proliferation: even authentic video and audio can now be credibly denied as synthetic. The existence of deepfake technology gives bad actors a plausible-deniability defense for real evidence against them.

Part V: Case Studies — ML Applied to Real Problems

Case 1: GDELT and Global Event Detection

The Global Database of Events, Language, and Tone (GDELT) is one of the largest open-source event databases in the world. It monitors print, broadcast, and web news in over 100 languages from nearly every country, extracting named entities, events, locations, and sentiment in near real-time using NLP and machine learning.

What it produces: Every 15 minutes, GDELT updates a record of events — conflicts, protests, diplomatic statements, natural disasters — tagged by actors, location, date, and “tone” (positive or negative sentiment). The database now covers events from 1979 to present across over 300 event categories.

Intelligence applications:

What GDELT enables

  • Detecting protest surges and civil unrest at sub-national scale before they appear in traditional media
  • Monitoring shifts in bilateral diplomatic tone between countries over time
  • Mapping conflict event intensity across geographic regions with lag of minutes rather than days
  • Cross-linguistic monitoring of narratives in state-controlled media

What GDELT cannot do

  • Confirm whether reported events actually occurred (it codes media reports, not ground truth)
  • Distinguish between genuine events and coordinated information operations in media
  • Capture events that receive no media coverage (a persistent sampling bias toward wealthier, media-accessible regions)
  • Provide individual-level data on actors or fine-grained causal analysis
NoteGDELT and the Data Science Workflow

GDELT is an excellent illustration of the data science pipeline in practice. The raw data (billions of event records) must be filtered (selection), processed (dates, locations, and event codes need normalization), explored (EDA to understand coverage patterns), and analyzed against a research question before it yields intelligence value. It is not a finished intelligence product — it is a starting point.

Case 2: Algorithmic Bias in Predictive Policing

In 2016, ProPublica published an investigation into COMPAS (Correctional Offender Management Profiling for Alternative Sanctions), a commercial risk assessment algorithm used by courts in several U.S. states to estimate defendants’ likelihood of re-offending. The investigation found that:

  • Black defendants were nearly twice as likely as white defendants to be falsely flagged as high-risk
  • White defendants were more likely to be incorrectly labeled low-risk
  • The algorithm’s overall accuracy was barely better than random chance for predicting re-offending

The algorithm’s developers (Northpointe) countered that the model was equally accurate across racial groups when measured differently — a genuine statistical dispute that reveals that fairness has multiple mathematical definitions that cannot all be satisfied simultaneously when base rates differ across groups.

Why this matters for intelligence analysis:

WarningAlgorithmic Bias is a Data Science Problem, Not Just an Ethics Problem

The COMPAS case illustrates that training data encodes historical patterns — including historical patterns of unequal treatment. A model trained on historically biased data will reproduce those biases at scale and with apparent mathematical authority. Any predictive model used in an intelligence or security context (threat scoring, travel screening, financial monitoring) faces the same structural risk. Knowing this is part of being a data-literate analyst.

Case 3: NSA SKYNET — ML-Based Targeting

This case is examined in detail in Part V of this lecture, in the context of ML deployment in intelligence practice. The core data science lesson: inadequate problem specification, small and uncertain training data, and insufficient validation can produce a model that is confidently wrong — with life-or-death consequences. See Part V for the full methodological analysis.

Part VI: Ethics and Limits in Data Science

The Problem of Predictive Systems in High-Stakes Contexts

Data science is a powerful tool. It is also increasingly being deployed in contexts — criminal justice, immigration enforcement, military targeting, intelligence analysis — where errors have serious consequences for individuals who may have no recourse.

Feedback Loops
When a predictive model influences decisions that generate new data, which then trains the next model, biases can amplify over time. Example: if a crime prediction model directs more police to certain neighborhoods, more arrests occur there, reinforcing the model’s prediction — independent of whether crime is actually higher there.
Proxy Discrimination
Using variables that are statistically correlated with protected characteristics (race, religion, national origin) can produce discriminatory outcomes even when those characteristics are not directly included. Example: zip code is correlated with race in the United States; a model using zip code inherits racial disparities without explicitly including race.
Model Opacity
Complex ML models (deep neural networks, gradient boosting ensembles) often cannot explain why they produced a specific output. When decisions affecting individuals are made on the basis of such models, due process and accountability are compromised. The IC faces similar constraints when algorithm outputs inform lethal operations.
Data Sovereignty
Who owns the data used to train a model? When the IC uses commercial data (social media, location data from data brokers, commercial satellite imagery), questions arise about what legal authorities govern collection, retention, and use — and about the rights of individuals whose data is used without their knowledge or consent.
Automation Bias
The tendency of human decision-makers to defer to algorithmic outputs even when they have contradicting information. Studies show that analysts and clinicians often override their own correct intuitions when an algorithm suggests otherwise. In intelligence, this means that human review of AI outputs is meaningless if reviewers are not actually evaluating the output critically.
TipDiscussion: Where Should Humans Stay in the Loop?

Consider each of the following intelligence-adjacent applications of ML. For each: what are the consequences of a false positive? A false negative? Should a human be required to approve each decision, review a sample, or can the system act autonomously?

  1. Flagging financial transactions as potentially terrorist-financing
  2. Classifying satellite imagery objects as military vehicles
  3. Ranking social media accounts as likely bot/inauthentic accounts
  4. Scoring individuals’ travel visa applications for fraud risk
  5. Predicting the probability of conflict onset in a specific country within 6 months

Part VII: Machine Learning in Intelligence Practice

From Research to Operations

Machine learning has moved from research demonstrations to operational deployment across the intelligence community and military. The following cases illustrate both the capabilities and the significant analytical and ethical challenges.

Case 1: Project Maven — Automated Imagery Exploitation

Project Maven (formally the Algorithmic Warfare Cross-Functional Team) was established by the U.S. Department of Defense in 2017 to apply ML to the problem of processing video footage from unmanned aerial systems (UAS). The scale of the problem was enormous: DoD was generating vastly more video than human imagery analysts could review in anything approaching real time.

Technical approach: Convolutional neural networks trained on labeled imagery to identify vehicles, people, structures, and activities of military interest. The system flagged frames of interest for human review rather than making autonomous targeting decisions.

The Google controversy: The contract was initially performed by Google, which triggered a significant employee backlash. Over 3,000 Google employees signed a letter objecting to the company’s involvement in weapons systems. Google declined to renew the contract in 2018, citing ethical concerns. The project subsequently moved to other contractors.

Analytical implications: Project Maven illustrated both the genuine capability of ML for imagery exploitation and the complex organizational and ethical dimensions of deploying ML in lethal operations contexts. The “human-on-the-loop” model — where humans review ML outputs and make final targeting decisions — remains contested because human reviewers may not be able to meaningfully evaluate the ML system’s confidence level.

Case 2: Adversarial Machine Learning

ML models can be deliberately attacked to cause misclassification. Adversarial examples — inputs constructed with small, carefully designed perturbations that are imperceptible to humans but cause ML models to misclassify with high confidence — are a fundamental vulnerability of current deep learning architectures.

Intelligence applications of adversarial ML:

Offensive (adversary uses against IC)

  • Generate satellite imagery containing adversarial perturbations that cause automated change detection to miss facility expansions
  • Create facial images that defeat recognition systems while appearing normal to human reviewers
  • Craft SIGINT content designed to cause NLP classifiers to misclassify message intent
  • “Poison” training data by inserting labeled examples that introduce systematic misclassification for specific targets

Defensive considerations

  • ML models used in consequential decision-making must be evaluated for adversarial robustness, not just clean-data accuracy
  • Adversarial testing (“red-teaming” the ML system) is a required component of responsible deployment
  • Human review of ML outputs is partially a defense against adversarial attacks — humans are robust to the specific perturbations that fool CNNs
  • Data provenance matters: poisoned training data is the hardest adversarial attack to detect and mitigate
WarningModel Opacity and Accountability

Explainability — the ability to understand why a model made a specific prediction — is a fundamental requirement for responsible ML deployment in intelligence contexts. A model that cannot explain its predictions cannot be reviewed for adversarial manipulation, cannot be audited for bias, and cannot support the “human-on-the-loop” accountability requirement. The tension between model complexity (which increases predictive accuracy) and interpretability (which decreases with complexity) is one of the central unsolved problems in responsible AI deployment. See Rudin (2019) in the readings below.

Case 3: NSA SKYNET — Targeting by ML

Classified NSA documents released by Edward Snowden included references to a program called SKYNET that applied ML to metadata (call records, device identifiers, location data) to identify suspected Al-Qaeda couriers in Pakistan. The program reportedly used a random forest classifier trained on behavioral metadata features.

A 2016 analysis by Ars Technica and Human Rights Data Analysis Group researchers examined the publicly available training data and model description and identified significant methodological concerns:

  • The model was trained on extremely small numbers of confirmed terrorist examples — a classic small-sample problem that tends to produce overfit models with poor out-of-sample accuracy
  • The feature engineering included features that conflate innocent behavior with terrorism indicators (e.g., “uses multiple SIM cards” describes both operational security tradecraft and ordinary behavior in regions with unreliable mobile networks)
  • The reported false positive rate — if the model was applied at scale to the Pakistani mobile phone population — would have flagged thousands to tens of thousands of innocent individuals

The consequence: Individuals flagged by SKYNET’s output were reportedly placed on target lists. The drone strike program in Pakistan was documented to have killed individuals based on pattern-of-life and metadata analysis rather than positive identification. If the SKYNET model had the false positive rates suggested by the independent analysis, some of those strikes may have been based on a flawed ML prediction.

Part VIII: Natural Language Processing in Intelligence Analysis

Why NLP Matters

A substantial fraction of intelligence-relevant information is text: intercepted communications, media reporting, social media posts, diplomatic cables, technical documents, extremist forum content, scientific publications, financial reports. NLP is the subfield of ML that enables computational analysis of this unstructured text.

Core NLP tasks relevant to intelligence:

Named Entity Recognition (NER)
Automatically identifying and classifying named entities in text: persons, organizations, locations, dates, facilities, weapons systems, financial values. Enables automated extraction of relational data from unstructured text at scale.
Topic Modeling
Unsupervised discovery of latent thematic structure in a corpus. Latent Dirichlet Allocation (LDA) and similar methods identify groups of co-occurring words that represent coherent topics. Applications: monitoring foreign media coverage, identifying thematic shifts in extremist forum content.
Machine Translation
Neural machine translation (as in Google Translate and equivalent classified systems) enables rapid translation of foreign-language sources. Limitations: specialized vocabulary (weapons systems, technical terminology, regional dialects) often translates poorly; irony, sarcasm, and culturally specific meaning are poorly handled.
Summarization
Automatic extraction or generation of key points from longer documents. Extractive summarization selects important sentences; abstractive summarization generates novel summary text. Quality varies significantly by domain and text type.
Large Language Models (LLMs)
Transformer-based models (GPT, Claude, Gemini, and classified equivalents) trained on vast text corpora that can answer questions, summarize, translate, generate, and analyze text through natural language interfaces. LLMs do not have access to real-time information without retrieval augmentation, and they generate plausible-sounding but sometimes factually incorrect text (“hallucinations”). Their use in classified environments is subject to significant access control and verification requirements.
